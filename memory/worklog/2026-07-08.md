## Built today: forecast verification (feature #6, mockup approved 07-07)
1. **`pipeline/verify_airnow.py`** — nightly pairing script (Kamiak, aqf env):
   - `--date YYYYMMDD` (default yesterday, local PDT) / `--backfill YYYYMMDD`
   - Fetches AirNow hourly obs for the local day (UTC window, `/aq/data`, dataType=B), key from `AIRNOW_API_KEY` env or `~/.airnow_env`
   - Pairs each obs hour/site with archived cycles: lead day 1 = same-day cycle, 2, 3 = older; nearest grid cell via Web-Mercator math matching viewer's `toIJ`
   - Stores SUFFICIENT STATISTICS (n, Σo, Σf, Σo², Σf², Σof) — page computes bias/RMSE/r for any filter combo. Two granularities: per (site, species, lead-day) per day [`rec`], per (species, lead-hour 0-71) domain-wide per day [`lh`]
   - Output: `web_out/verification/history/<day>.json` (kept forever) + `summary.json` (rolling 90 days, published)
   - State from FullAQSCode FIPS prefix (53=WA 41=OR 16=ID 30=MT 32=NV 06=CA 56=WY 49=UT, else OTH e.g. BC)
2. **`pipeline/verify.html`** — standalone page at `/verify.html`: filters (species/window 7-30-90d/region/lead), KPI cards, scatter of site-day means w/ 1:1, bias-by-lead-hour line, daily domain-mean time series (obs dots vs fcst line), worst-RMSE site table (min 8 pairs). Fetches `data/verification/summary.json`; friendly empty state if missing.
3. **Wiring**: `postprocess_and_publish.sh` runs verify for yesterday between run_post and publish (non-fatal, own conda activation); `publish_cloudflare.sh` copies verify.html + summary.json (both no-cache); viewer sidebar link "Forecast verification →" under Overlays.
